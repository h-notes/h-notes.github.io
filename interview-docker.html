有关 Dcoker 容器主进程指定   NOTE

[2020-04-05 日 11:02]

https://docker_practice.gitee.io/us-en/image/dockerfile/entrypoint.html

  1. 首先启动容器，实际上就是在启动主进程。容器为主进程提供了独立的运行环境。
  2. 容器以具体的本地镜像为基础进行启动。因此镜像在构建阶段需要指定进程的启动运行命令。

  3. 有两条 Dockerfile Command 与之相关：ENTRYPOINT, CMD

    3.1 CMD 用来指定镜像在被启动时的默认的程序以及参数，该程序以及参数在 docker run 时可以被替换。 3.2 ENTRYPOINT 的目的和 CMD 一样，其指定的主进程命令以及参数，也可以在 docker run 时被替换，但是需要 –entrypoint 来指定。

    当指定了 ENTRYPOINT 以后，CMD 的意义就发生了变化：不是直接运行 CMD 指定的命令，而是将 CMD 的内容作为参数传给 ENTRYPOINT。

    ENTRYPOINT CMD

  4. ENTRYPOINT 应用场景

    4.1 用来指定基本的主进程命令以及参数，而其他参数可以在 docker run 时动态传递。

    4.2 有些时候程序在启动之前需要做一些其他配置，初始化的工作。

    这类复杂的主进程启动配置可以使用如下方法：

    写一个 entrypoint.sh 脚本，该脚本作为实际的启动命令，用 ENTRYPOINT 指定。然后，动态的 CMD 参数就传给了 entrypoint.sh，让其根据参数做具体的处理。

Dockerfile VOLUME command   NOTE

[2020-04-05 日 11:41]

使用 VOLUME /data 可以指定匿名卷，容器主进程运行时任何向 /data 写入的数据都不会记录在容器存储层（容器动态层，随着容器的退出而自动消亡）。

可以用 docker run -d -v mydata:/data xxx，将 /data 实名化，将其挂载至 host mydata 路径

docker network   NOTE

[2020-04-08 三 21:14]

port mapping

使用 hostPort:containerPort 格式本地的 5000 端口映射到容器的 5000 端口，可以执行

docker run -d -p 5000:5000 training/webapp python app.py

容器互联

  1. 新建网络 docker network create -d bridge my-net
  2. 连接容器 docker run -it –rm –name busybox1 –network my-net busybox sh docker run -it –rm –name busybox2 –network my-net busybox sh

下面通过 ping 来证明 busybox1 容器和 busybox2 容器建立了互联关系。

/ # ping busybox1 PING busybox1 (172.19.0.2): 56 data bytes 64 bytes from 172.19.0.2: seq=0 ttl=64 time=0.064 ms 64 bytes from 172.19.0.2: seq=1 ttl=64 time=0.143 ms

数据卷 Volume 如何使用

[2020-04-04 六 17:25]

通常容器的存储层会随着容器的消亡和消失，因此需要通过 Volume 将容器封装的应用产生的需要持久化的数据存储在 Host 或者其他网络文件服务上。

数据卷

  1. 创建数据卷

docker volume create my-vol

  1. 下面创建一个名为 web 的容器，并加载一个 数据卷 到容器的 /webapp 目录。

$ docker run -d -P \ –name web \

–mount source=my-vol,target=/webapp \ training/webapp \ python app.py

如果需要在删除容器的同时移除数据卷, 可以在删除容器的时候使用 docker rm -v 这个命令 无主的数据卷可能会占据很多空间，要清理请使用以下命令: docker volume prune

挂载主机目录

$ docker run -d -P \ –name web \

–mount type=bind,source=/src/webapp,target=/opt/webapp \ training/webapp \ python app.py

Dockerfile CMD command   NOTE

[2020-04-11 六 18:15]

CMD 指令就是用于指定默认的容器主进程的启动命令的

格式

  • shell 格式：CMD <命令>
  • exec 格式： CMD ["可执行文件"，"参数1", "参数2"…]

  • 参数列表格式：CMD ["参数1", "参数2"…]。在指定了 ENTRYPOINT 指令后，用 CMD 指定具体的参数。

    shell 格式在实际中被执行时，会被解释成 CMD ["sh", "-c", "echo $HOME"]
