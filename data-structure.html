HyperLogLog   NOTE

[2020-01-13 一 20:43] Organization

  • resource
  • A HyperLogLog is a probabilistic data structure.

It counts the number of distinct elements in a list. But in comparison to a straightforward way of doing it (having a set and adding elements to the set) it does this in an approximate way.

  • The HyperLogLog data structure can be used in order to count unique elements in a set using just a small constant amount of memory, specifically 12k bytes for every HyperLogLog (plus a few bytes for the key itself).

Probabilistic data structures will give you memory-efficient, faster result with a cost of providing a ‘probable’ result instead of a ‘certain’ one。

Bloom filter

good resource.

Bloom filter is all about testing Membership in a set.

  • 使用一个 bit-vector(or integer-vector)以及多个 hash function, 对每个 key 用 hash function 求值，然后将每个结果作为index在 bit-vector 中 set。
  • test(key)，操作使用上述相同的操作，只要有一个 hash value index 未被 set, 则该 key 一定不存在, 如果都被 set, 则改 key 可能存在。可能性取决于 vector 的大小以及 hash function 的数量。
